# 設定ファイルの仕様ファイルの形式はYAMLであり，設定を階層的に定義している．最上位ハッシュのキーは，実機とシミュレーションで共通の`commnon`，実機の設定をする`real`，シミュレーションの設定の`sim`である．それぞれのキーの下には，ネームサーバ，RTCの一覧，RTCの起動コマンド，RTC 間の接続が記述されている．`sim`にはChoreonoidのプロジェクトの設定なども含まれる．## common実機とシミュレーションで実行される共通部分．RTCが複数あっても可．- userComps  - command:     - 実行するコマンド    - 例 : KobukiController\build\src\Debug\KobukiControllerComp.exe  - rtc:     - RTC名．以下の```- rtc: ```のタグは全てRTC名を記述する．    - 例 : KobukiController0.rtc## real実機側で必要な設定部分．リモートPCのIPアドレス，リモートRTC名，confファイル，RTC接続について記述する．- server:  - リモートPCのIPアドレス  - 例 : 192.168.11.100- userCompConf:  - 実機用のconfファイル．詳細は仕様書を参照．- remoteComps:  - リモートRTC名．- connections:  - rtconと同じ記述をするが，パスが不必要でRTC名とポート名だけでよい．  - 例 : KobukiRTC0.rtc:bumper KobukiController0.rtc:bumper## simシミュレーションに必要な設定部分．realと被っている部分は省略する．- supportComps:記述方法は```common```の```userComps```と同じ要領．  - command:     - 実行するコマンド  - rtc     - 支援RTC名- choreonoidCommand:  - コレオノイドの実行コマンド  - 例 : C:\Choreonoid\bin\choreonoid.exe- projects:コレオノイドのプロジェクトファイルとその名称  - path:    - プロジェクトファイル    - KobukiForChoreonoid\kobuki.cnoid  - label:    - シミュレーション環境に適切な任意の名称- choreonoidComps:コレオノイドの機能のBodyRTC名  以下は設定ファイルの具体的な階層．  ```common:  userComps:    - command: 実行コマンド1      rtc: RTC名1    - command: 実行コマンド2      rtc: RTC名2    #任意回繰り返し可能real:  server: ネームサービスの根となるアドレス  userCompConf: conf ファイル  remoteComps:    - rtc:RTC名1    - rtc:RTC名2    #任意回繰り返し可能      connections:    - 出力RTC名1:出力ポート名1 入力RTC名1:入力ポート名1    - 出力RTC名2:出力ポート名2 入力RTC名2:入力ポート名2  #任意回繰り返し可能sim:  server: ネームサービスの根となるアドレス  userCompConf: confファイル  supportComps:    - command: 実行コマンド1      rtc: RTC名1    - command: 実行コマンド1      rtc: RTC名2    #任意回繰り返し可能      choreonoidCommand:choreonoid実行コマンド  projects    - path: プロジェクトファイルパス1      label: コンボボックスに表示する名称1    - path: プロジェクトファイルパス2      label: コンボボックスに表示する名称2    #任意回繰り返し可能  choreonoidComps:    - rtc: RTC名1    - rtc: RTC名2    #任意回繰り返し可能  connections:    - 出力RTC名1:出力ポート名1 入力RTC名1:入力ポート名1    - 出力RTC名2:出力ポート名2 入力RTC名2:入力ポート名2    #任意回繰り返し可能```
